# Writing Conventions

Rules applied to Markdown content generated by research-note.

## Universal Rules

- **One sentence per line**: Each sentence on its own line, no mid-sentence breaks
- **No period in list items**: Never end list items with a sentence-ending period
- **Cite sources**: Use footnotes `[^1]: https://...` for web-searched information
- **No AI hype**: Avoid exaggerated phrases ("groundbreaking", "revolutionary", "game-changing")

## Analytical Insight

Insights emerge naturally from connecting new information to existing knowledge — they should not be forced.
The depth of understanding built in Workflow Step 2a (reading existing notes) determines note quality.

When insights arise naturally, add a sentence from one of these perspectives:

- **Comparison with existing concepts**: "X differs from Y in that..."
- **Articulating the essence**: Abstract from specific examples to explain why something is effective or important
- **Limitations and trade-offs**: Show where something doesn't work, not just where it does

Unlike factual claims requiring footnotes, insights are the author's analysis and can be written without citations.
However, keep fact and opinion clearly distinguishable — placing an insight sentence right after its supporting fact works well.
If no insight emerges naturally, omit it — a formulaic observation is worse than none.

## Annotation Format

Formatting for notes, warnings, and supplementary information.

- Warnings/cautions: `> [!WARNING]` / `> [!CAUTION]`
- Notes/tips: `> [!NOTE]` / `> [!TIP]`
- Use only the 5 GitHub-supported types (NOTE, TIP, IMPORTANT, WARNING, CAUTION)
- Short inline notes where blockquotes are structurally impossible (e.g., table cells): use a brief marker like `*`
- Long supplementary content (5+ lines): `<details>`

## Citation Format

Every web-sourced factual claim must have a footnote citation.

```markdown
Anthocyanins change color depending on pH[^1].

[^1]: https://example.com/anthocyanin-ph
```

### Footnote Deduplication

Consolidate identical URLs into a single footnote number, referenced from multiple locations.

```markdown
<!-- Good -->
Feature A was added[^1]. Feature B was also announced[^1].

<!-- Bad -->
Feature A was added[^1]. Feature B was also announced[^2].
[^1]: https://example.com/release
[^2]: https://example.com/release
```

### Citation Integrity

- Citing a URL without reading its content is prohibited
- Use Task subagents with `fetch-page.js` to verify content and confirm it supports the claim

## Headless Chrome (fetch-page.js)

Use headless Chrome via `fetch-page.js` for URL content retrieval.

```bash
node .claude/skills/research-note/scripts/fetch-page.js <URL>
node .claude/skills/research-note/scripts/fetch-page.js <URL> --selector "article"
```

- Output is Markdown (preserves heading and list structure)
- Takes several seconds due to Chrome launch (`--timeout` option available)
- Run via Task subagents to keep the main context compact
